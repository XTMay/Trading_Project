1 系统概览

盈再表是一个基于 Excel VBA 的多市场股票财务分析系统，原始文件为 盈再表250723.xlsm。 系统覆盖五大股票市场，具备数据抓取、财报分析、投资组合管理、绩效追踪及税务计算等功能。

注记文件编码说明

原始 .bas / .cls 导出文件采用 Big5（繁体中文） 编码，已批量转换为 UTF-8。 原始备份保存于各目录下的 _backup_big5/ 文件夹中。

3 分层架构

系统没有显式的架构设计，但根据代码职责可归纳为五层：

3.1 各层详细说明

3.1.1 入口层（事件触发）

系统的所有执行均由 Excel 事件驱动，无独立启动入口。

3.1.2 UI 层

重要无传统 UI 层

UserForm1~5.frm 全部为空壳（代码区无逻辑，控件定义在二进制 .frx 中）。 系统的实际 UI 完全依赖 Excel Sheet 单元格交互——用户在 A2 输入股票代号即触发数据抓取。

3.1.3 业务逻辑层

3.1.4 数据抓取层

这是系统最核心、代码量最大的部分。每个 Module 对应一个市场，内部结构高度一致：

Macro{N}()
  1. UnprotectSheet()          ← 解锁工作表
  2. 清除旧数据区域             ← Range().Clear
  3. 循环抓取多种财务数据        ← HTTP 请求（5~11 种）
  4. 解析 HTML/JSON             ← DOM 解析 / JsonConverter
  5. 写入对应列                 ← Cells / Range 赋值
  6. ProtectSheet()             ← 重新锁定
  7. Beep                      ← 提示完成

5 关键发现与风险评估

5.1 架构特征

提示设计模式
  • 1 Sheet = 1 Module 的对应关系，结构清晰、一致
  • 所有市场 Module 遵循相同的执行模板（解锁 → 清除 → 抓取 → 写入 → 锁定）
  • 通过 ProtectSheet / UnprotectSheet 实现简单的”事务”控制
特征 说明
架构模式 事件驱动 + 过程式，无 OOP 设计
Sheet-Module 映射 严格 1:1（Sheet1↔︎Module1, …, Sheet5↔︎Module5）
类封装 无。仅用 Public Type baseInfo 做简单结构体
UserForm 全部空壳，实际 UI 完全依赖 Excel 单元格
错误处理 大量 On Error Resume Next，出错时静默跳过
配置管理 硬编码在代码中（URL、列号、数据源地址）

5.2 风险点

警告高风险项

以下问题在系统迁移或维护时需重点关注。

5.2.1 1. 数据源脆弱性

系统依赖 10+ 个外部网站的 HTML 结构进行解析。任一网站改版（DOM 结构变化）都会导致对应市场的数据抓取完全失败。

受影响的关键数据源：

  • MoneyDJ、TWSE、TPEX（台股）
  • MarketWatch、WSJ（美股/全球）
  • Yahoo Finance（全市场历史价格）
  • AAStocks（港股/中股）
  • Sina Finance（中股）

5.2.2 2. 重复代码

HTTP 连接函数（ConnectXMLHTTP、ConnectWinHttp、ConnectMarketWatch）在 多个 Module 中重复定义，未集中到 shared_modules。维护时需同步修改多处。

5.2.3 3. 静默错误

On Error Resume Next 在整个代码库中普遍使用，导致：

  • 网络请求失败时不会提示用户
  • 数据解析错误时写入空值或错误值
  • 难以定位问题根源

5.2.4 4. 平台兼容性

功能 Windows macOS
MSXML2.XMLHTTP Y N
WinHttp.WinHttpRequest Y N
InternetExplorer.Application Y N
QueryTables.Add Y 部分
Big5 编码 VBA 编辑器 Y 乱码
注意

当前数据抓取层几乎完全依赖 Windows COM 组件（MSXML2、WinHttp、IE），无法在 macOS 上运行。

5.2.5 5. 命名规范

  • 所有宏使用 Macro{N} 命名（Macro1~Macro30），无语义化命名
  • 变量大量使用单字母（f, dc, cd$）
  • 注释为繁体中文，部分因编码转换存在缺字

6 建议的分析顺序

针对后续深入逆向工程或系统重写，建议按以下优先级推进：

阶段 步骤 目标文件 目的
0 阅读安裝 + 介紹 Sheet Excel 工作簿 理解作者意图、使用方法、系统要求
1 公共基础 shared_modules.bas 掌握全局变量、类型定义、findAll、Stocklist、保护机制
2 启动入口 ThisWorkbook_7.cls 理解工作簿打开时的初始化流程
3 核心路径 Sheet1.cls + Module1.bas 最复杂的实现（台股11种数据源），理解它即理解整个系统模式
4 对比分析 Module2.bas 美股实现，引入 Yahoo Finance + JSON 解析，与台股对比差异
5 快速扫描 Module3~5.bas 架构相同，仅数据源不同，确认差异点
6 业务串联 Module7.bas 组合管理如何串联所有市场 Module
7 绩效算法 Module8.bas XIRR + 巴菲特距离公式
8 独立功能 Module6.bas + Module9.bas 选股筛选 + 税务计算
9 第三方库 JsonConverter.bas 确认版本与兼容性
