1 事件检测结果

对项目中所有 .cls（Workbook / Sheet 对象）和 .frm（UserForm）文件进行扫描，检测标准 VBA 事件处理器。

1.1 事件存在性总表

重要关键发现
  • 系统只有 8 个事件处理器，全部集中在 2 个文件中（ThisWorkbook_7 + Sheet1~5）
  • UserForm1~5 全部为空壳：.frm 文件代码区无任何 Sub/Function
  • 没有任何按钮点击事件：UI 交互完全依赖单元格编辑触发 Worksheet_Change
  • Sheet6~9、Sheet01、Worksheet___16、ThisWorkbook_16 均无代码

2 系统启动入口

系统有 3 个自动触发入口 和 1 类手动触发入口。

2.1 自动触发入口（打开工作簿时）

2.1.1 入口 ① — Workbook_Open

属性 值
文件 ThisWorkbook_7.cls:10
修饰符 Private Sub
触发时机 工作簿打开时自动执行
错误处理 On Error Resume Next（静默）

第一层调用：

调用目标 来源 作用
Delete_Connection_XL2003 shared_modules.bas 删除旧版 Excel 数据连接
QueryTablesDelete("all") ThisWorkbook_7.cls:44 (本文件) 清除所有工作表的 QueryTable

直接设置的 Application 属性：

Application.ErrorCheckingOptions.BackgroundChecking = False
Application.CommandBars("External Data").Visible = False
Application.AutoRecover.Enabled = False
Application.AskToUpdateLinks = False
Application.MaxChange = 0.001
Application.AutoCorrect.* = False  (6项全关)
ActiveWorkbook.UpdateRemoteReferences = False
ActiveWorkbook.PrecisionAsDisplayed = False
ActiveWorkbook.SaveLinkValues = False

2.1.2 入口 ② — Auto_open

属性 值
文件 shared_modules.bas:242
修饰符 Sub（Public，VBA 自动识别）
触发时机 工作簿打开后自动执行

第一层调用： 无函数调用。仅向两个 Sheet 的 ZZ1 单元格写入双引号字符。

Sub Auto_open()
    Sheets("台股").[zz1] = """"
    Sheets("美股").[zz1] = """"
End Sub
注记

Workbook_Open 和 Auto_open 的执行顺序：Excel 先执行 Workbook_Open，再执行 Auto_open。两者共同构成完整的启动序列。

2.2 用户交互入口（Worksheet_Change）

这是系统的主交互路径——用户在某个市场 Sheet 的 A2 单元格输入股票代号，触发数据抓取。

2.2.1 Sheet1（台股）— 最复杂的入口

属性 值
文件 Sheet1.cls:10
触发条件 编辑台股 Sheet 的 A1 或 A2 单元格
Guard 条件 排除全行选中、Column > 1、Row > 2

第一层调用链：

序号 调用目标 所在模块 作用
1 findAll(text) shared_modules.bas 在股票数据库中查找代号/名称
2 Stocklist() shared_modules.bas 若未找到，刷新台股代号清单
3 findAll(text) shared_modules.bas 重试查找
4 Macro1() Module1.bas 核心：台股11种财务数据抓取
5 ProtectSheet(ActiveSheet) shared_modules.bas 锁定工作表

2.2.2 Sheet2~5 — 统一模式

Sheet2~5 采用完全相同的极简结构：

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Address <> "$A$2" Then Exit Sub   ' 仅监听 A2
    Macro{N}                                     ' 直接调用对应 Macro
    Call ProtectSheet(ActiveSheet)               ' 锁定
    Beep                                         ' 提示音
End Sub
Sheet 文件 Guard 条件 第一层调用
Sheet2（美股） Sheet2.cls:10 Target.Address <> "$A$2" Macro2 → ProtectSheet
Sheet3（港股） Sheet3.cls:10 Target.Address <> "$A$2" Macro3 → ProtectSheet
Sheet4（中股） Sheet4.cls:10 Target.Address <> "$A$2" Macro4 → ProtectSheet
Sheet5（全球） Sheet5.cls:10 Target.Address <> "$A$2" Macro5 → ProtectSheet
提示设计模式

Sheet1 比 Sheet2~5 多了 findAll + Stocklist 的查找逻辑，因为台股支持通过股票名称（而非仅代号）触发查询。Sheet2~5 仅支持直接输入代号。

5 异常发现

分析过程中发现以下需要注意的问题：

警告JsonConverter.bas 文件异常

JsonConverter.bas 不是第三方 JSON 解析库，而是 Module8.bas（绩效计算）的重复副本。 如果代码中有 JsonConverter.ParseJson 调用，运行时将找不到该函数。 需确认：(a) 实际的 JSON 解析功能在哪里？(b) 是否已内联到 Module2 中？

6 结论：系统从哪里开始？

一句话总结：

系统通过 Workbook_Open + Auto_open 完成静默初始化，随后进入等待状态。 用户在任一市场 Sheet 的 A2 单元格输入股票代号，触发 Worksheet_Change → Macro{N} 执行数据抓取。 其他高级功能（组合管理、绩效、税务）均为手动调用的独立宏。
